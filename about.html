About the Project

 

APIs and Frameworks Used

  • Vue.js

    • Vue.js is an open-source modelview viewmodel front end JavaScript library for building user interfaces and single-page applications. It was created by Evan You, and is maintained by him and the rest of the active core team members.

  • Foundation CSS

    • Foundation is a free responsive front-end framework, providing a responsive grid and HTML and CSS UI components, templates, and code snippets, including typography, forms, buttons, navigation and other interface elements, as well as optional functionality provided by JavaScript extensions. Foundation is an open source project, and was formerly maintained by ZURB. Since 2019, Foundation has been maintained by volunteers.

  • Our Very Own Rest API ❤

    • Cooked up on our very own machines, our rest_server.mjs api is seriously (not) one of a kind. It allows us to easily talk back and forth with our database, we can use it to delete, insert, and fetch data. Without this, our app would not be able to load crime data!

  • Leaflet

    • Leaflet is the leading open-source JavaScript library for mobile-friendly interactive maps. Weighing just about 42 KB of JS, it has all the mapping features most developers ever need. Leaflet is designed with simplicity, performance and usability in mind. It works efficiently across all major desktop and mobile platforms, can be extended with lots of plugins, has a beautiful, easy to use and well-documented API and a simple, readable source code that is a joy to contribute to.

  • Nominatim

    • Nominatim uses OpenStreetMap data to find locations on Earth by name and address (geocoding). It can also do the reverse, find an address for any location on the planet. The latest release is 4.3.2 and comes with tons of documentation on how to use the API. Some key supporters include OpenCage, Graphhopper, and Komoot. This API makes it super easy to convert back and forth between longitude/latitude coordinates and addresses.



Six Intersting Things Using the Application

  1. Creating the markers for the map was a pain in the butt.
  2. Leaflet has a ton of cool functions/features you can use, but the list is so vast I'd be amazed to see what leaflet CAN'T do.
  3. Vue.js has a ton of sweet in-app features that made grabbing stuff from API's super easy.
  4. Most of the crime in St. Paul occurs right near the capital.
  5. The latitude/longitude coordinates do not change a ton as you traverse around the map, meaning you have to be ultra precise in how you search.
  6. The northeast portion of St. Paul tends to be safest, as the number of crimes drops in those neighborhoods compared to others.
